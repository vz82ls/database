Unit-1 : Database

1. Data and Information

Data: Data are raw facts and figures which are unorganized and donâ€™t provide meaningful information unless processed. Example: Ram, 17, Pokhara, 18

Information: Processed data which conveys meaningful results. Example: Ram is 17 years old and lives in Pokhara 18.

Problems with unorganized data include redundancy, slow retrieval, and poor security. To solve these, databases are used.

2. Database & Flat File System

A Database is a structured collection of related data. It allows easy access, management, and updating of data. Examples: dictionary, phonebook, marks ledger.

Flat File System limitations include:

  • Redundant data
  • Slow processing
  • Low security
  • Difficult sharing
3. DBMS

DBMS is a software for managing databases. It allows storage, retrieval, and updating of data. Examples: MS Access, Oracle, MySQL.

Advantages:

  • Data sharing
  • Less redundancy
  • Security
  • Data backup

Disadvantages: Expensive, needs training, requires backups.

4. Database Models

Hierarchical Model: Tree structure, parent-child relationship.

Network Model: Graph structure, supports many-to-many relationships.

Relational Model: Tables with rows and columns (RDBMS).

5. Centralized vs Distributed Database

Centralized: All data stored in one location. Easy to maintain but prone to failure.

Distributed: Data stored in multiple locations. More secure and faster but costly.

6. SQL (Structured Query Language)

SQL is used to interact with relational databases. Developed by IBM in the 1970s.

  • DDL: Create/structure tables. e.g. CREATE TABLE
  • DML: Manipulate data. e.g. INSERT, DELETE
  • DCL: Control access. e.g. GRANT
7. ER Model

Entity: Real-world object (rectangle)

Attribute: Property of entity (ellipse)

Relationship: Association between entities (diamond)

8. DBA (Database Administrator)

A person with full control over DBMS. Responsibilities include security, updates, backups, training, and access control.

9. Normalization

Breaking complex tables into smaller ones to reduce redundancy. Involves:

  • 1NF: Eliminate duplicate columns
  • 2NF: Remove partial dependencies
  • 3NF: Remove non-key dependencies
